# موقع فعاليات محافظة درعا

دليل تثبيت وتشغيل بسيط لمشروع موقع فعاليات محافظة درعا.

## متطلبات

- Node.js مثبت على الجهاز
- npm متاحة من خلال Node.js

## خطوات تشغيل المشروع

1. افتح نافذة الطرفية (Terminal) في مجلد المشروع `hallo`.
2. ثبّت التبعيات:

```bash
npm install
```

3. شغّل الخادم:

```bash
npm start
```

4. افتح المتصفح وادخل إلى:

```text
http://localhost:3000
```

## بيانات الدخول الجاهزة

يمكنك تسجيل الدخول مباشرة باستخدام أحد الحسابين التاليين:

- يوجد حسابات مُعدة مسبقًا في النظام للمدير والمستخدم العادي، لكن يُفضل إنشاء حساب جديد لحماية الخصوصية.

## تجربة مستخدم جديد

يمكن لأي شخص إنشاء حساب جديد من صفحة الموقع عبر زر "إنشاء حساب".

## ملاحظات

- المشروع يستخدم قاعدة بيانات SQLite في المجلد `db/events.db`.
- يوفر النظام الآن حجز مقاعد تفاعلي (VIP، عام، خلفي) على خريطة مقاعد مبرمجة.
- وظيفة الدفع عبر Stripe تحتاج إعداد المتغير البيئي `STRIPE_SECRET_KEY`.

## إعداد Stripe محلياً

إذا أردت تفعيل صفحة الدفع محلياً، أنشئ ملف `.env` في جذر المشروع وأضف ما يلي:

```env
STRIPE_SECRET_KEY=sk_test_your_secret_key_here
APP_URL=http://localhost:3000
```

ثم شغّل الخادم مرة أخرى:

```bash
npm install
npm start
```

إذا لم تضع مفتاح Stripe، فإن `/api/events/:id/checkout` سيُرجع رسالة خطأ واضحة تفيد بأن Stripe غير مهيأ.
- يوفر المدير لوحة تحليلات لرصد مبيعات التذاكر، الإيرادات، ونسب الحضور لحظياً.
- تولد التذاكر رمز استجابة سريعة (QR) فريد لكل حجز، ويمكن تنزيلها كملف PDF.
- يمكن إعداد فعاليات هجينة (Hybrid Events) مع تذاكر افتراضية وحضور عن بُعد.
- إذا أردت إرسال المشروع لشخص آخر، أرسل المجلد كاملًا مع ملف `package.json` وملف `server.js`.
- لا تحتاج إلى إعداد مفتاح API للخرائط، حيث يتم استخدام OpenStreetMap وLeaflet.

## نشر المشروع على استضافة مجانية

يمكنك نشر المشروع بسهولة على خدمة مجانية تدعم Node.js مثل Render.

1. أنشئ حسابًا مجانيًا في `render.com`.
2. ارفع المشروع إلى مستودع GitHub.
3. في Render اختر `New` ثم `Web Service`.
4. اختر المستودع الخاص بالمشروع.
5. ضع إعدادات البناء:
   - Build Command: `npm install`
   - Start Command: `npm start`
6. اضغط `Deploy`.

> لقد تم تعديل `package.json` بحيث يستخدم `node server.js` مباشرة، وهذا يسمح للخدمة بتعيين المنفذ (`PORT`) تلقائياً.

### ملاحظات مهمة

- المشروع يستخدم SQLite محليًا في `db/events.db`.
- على خدمات الاستضافة المجانية قد تكون ملفات البيانات مؤقتة أو تُفقد إذا أُعيد نشر التطبيق.
- إذا أردت أن يعمل البريد الإلكتروني فعليًا في الإنتاج، فعليك إضافة متغيرات بيئة SMTP مثل `SMTP_HOST` و`SMTP_USER` و`SMTP_PASS`.
- يمكنك استخدام `APP_URL` لتعيين عنوان الموقع الكامل إذا احتاج التطبيق لذلك.

إذا تريد، أساعدك خطوة بخطوة في رفع المشروع على Render أو Replit بعد أن ترفع ملفاتك إلى GitHub.

## شرح الكود (موجز)

- `server.js`: الخادم الرئيسي المبني على Express. يحتوي على تعريف المسارات (API endpoints) للتسجيل، تسجيل الدخول، إدارة المستخدمين، الفعاليات، التذاكر، والتقارير التحليلية. يقوم بتهيئة قاعدة البيانات SQLite ويعرض الملفات الثابتة في `public/`.
- `db.js`: تهيئة ونقطة وصول بسيطة لقاعدة بيانات SQLite المستخدمة في المشروع.
- `lib/`:
   - `lib/server-utils.js`: مساعدات للمصادقة، توليد التوكن، وبناء روابط البريد الإلكتروني، وميدلوير `requireAuth` و`requireAdmin` لاستخدامها في المسارات.
   - `lib/pdf-utils.js`: منطق توليد ملفات PDF للتذاكر.
- `__tests__/`: مجموعة اختبارات تشغيلية ووحدوية مخصصة للمشروع. يوجد مشغّل مبسط `__tests__/run-tests.js` لتشغيل كل الاختبارات عبر `node` دون الحاجة لإطار اختبار خارجي.
- `public/`: ملفات الواجهة الأمامية (HTML, CSS, JS، أيقونات، صفحات SPA). تُقدّم مباشرة من الخادم.
- `uploads/`: مكان حفظ الملفات المرفوعة (صور الفعاليات، وسائط المستخدم).
- `db/`: يحتوي على ملف قاعدة البيانات SQLite (`events.db`) ونسخ البيانات المتعلقة بالتطبيق.
- ملفات مساعدة ومؤقتة:
   - `tmp-booking-test.js`, `tmp-dom-check.js`: سكربتات مؤقتة أو اختبارات سريعة تم استخدامها أثناء التطوير.

## ماذا تفعل كل مسارات الإدارة (`/api/admin/*`)

- `GET /api/admin/users`: يعرض قائمة المستخدمين (معلومات أساسية: `id`, `username`, `email`, `role`, `isActivated`). يتطلب صلاحية المدير.
- `POST /api/admin/users`: يسمح للمدير بإنشاء مستخدم جديد (يمكن تحديد `role` و`isActivated`).
- `PUT /api/admin/users/:id`: تعديل الحقول المصرح بها فقط (`role`, `isActivated`). لا يسمح بتعديل `username` أو `password` أو `email` عبر هذا المسار.
- `DELETE /api/admin/users/:id`: حذف مستخدم — لكن يحظر حذف حساب المدير نفسه.

## كيف تشغّل الاختبارات المحلية

لتشغيل جميع الاختبارات المضمنة (لا تحتاج Jest)، نفّذ:

```bash
node __tests__/run-tests.js
```

الاختبارات تغطي مساعدات الخادم (`lib/server-utils.js`)، توليد PDF، تدفقات المصادقة (تسجيل، نسيت كلمة المرور، إعادة التعيين)، ومسارات الإدارة مع حالات الأذونات.
